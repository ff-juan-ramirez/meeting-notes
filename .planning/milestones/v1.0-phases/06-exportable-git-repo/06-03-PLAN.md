    **Section: ## Prerequisites**
    - macOS with Apple Silicon (M1/M2/M3/M4)
    - Python 3.11+ (`brew install python` or pyenv)
    - ffmpeg (`brew install ffmpeg`)
    - BlackHole 2ch (`brew install blackhole-2ch`)
    - Ollama (`brew install ollama` then `ollama pull llama3.1:8b`)
    - Notion integration token (link to https://www.notion.so/my-integrations)

    **Section: ## Audio MIDI Setup** (D-15)
    - Step-by-step numbered instructions:
      1. Open Audio MIDI Setup (Spotlight: "Audio MIDI Setup")
      2. Click "+" at bottom-left, create "Multi-Output Device"
      3. Check both "BlackHole 2ch" and your speakers/headphones
      4. Set Multi-Output Device as system output (System Settings > Sound > Output)
    - ASCII art diagram showing signal routing:
      ```
      System Audio ──> BlackHole 2ch (device :1) ──┐
                                                     ├──> ffmpeg amix ──> recording.wav
      MacBook Mic  ────────────────── (device :2) ──┘
      ```
    - Note: system audio goes through both BlackHole (for capture) and speakers (for listening)

    **Section: ## Installation**
    ```bash
    git clone https://github.com/YOUR_USERNAME/meeting-notes.git
    cd meeting-notes
    pip install .          # or: pip install -e . (for development)
    meet init              # interactive setup wizard
    ```

    **Section: ## Quick Start**
    ```bash
    meet init              # first-time setup
    meet record            # start recording a meeting
    meet stop              # stop recording
    meet transcribe        # transcribe the last recording
    meet summarize         # generate notes + push to Notion
    meet list              # view all recordings
    ```

    **Section: ## Commands**
    Each command with description, flags, and example:

    - `### meet record` — Start recording system audio + mic. No flags. Example: `meet record`
    - `### meet stop` — Stop the active recording. No flags. Example: `meet stop`
    - `### meet transcribe` — Transcribe last or specified recording.
      - `--session UUID` — transcribe a specific session
      - Example: `meet transcribe` or `meet transcribe --session abc123`
    - `### meet summarize` — Generate structured notes from transcript.
      - `--template meeting|minutes|1on1` — note template (default: meeting)
      - `--session UUID` — summarize a specific session
      - Example: `meet summarize --template 1on1`
    - `### meet list` — Show all recordings with status.
      - `--status STATUS` — filter by status (not-transcribed, transcribed, summarized)
      - `--json` — output as JSON
      - Example: `meet list --status summarized`
    - `### meet doctor` — Check system prerequisites.
      - `--verbose` — show detailed check information
      - Example: `meet doctor --verbose`
    - `### meet init` — Interactive setup wizard.
      - Example: `meet init`

    Global flags (available on all commands):
    - `--quiet` — suppress progress output (for scripting)
    - `--version` — show version

    **Section: ## Configuration**
    - Config file: `~/.config/meeting-notes/config.json`
    - Show example config JSON:
      ```json
      {
        "version": 1,

          "system_device_index": 1,
          "microphone_device_index": 2
        },
        "whisper": {
          "language": null
        },
        "notion": {
          "token": "ntn_...",
          "parent_page_id": "your-page-id"
        }
      }
      ```
    - Explain: `language: null` = auto-detect, or set to `"en"`, `"es"`, etc.
    - Data stored in: `~/.local/share/meeting-notes/`

    **Section: ## Troubleshooting**
    - "No audio captured": Check Audio MIDI Setup — BlackHole must be in Multi-Output Device
    - "ffmpeg not found": `brew install ffmpeg`
    - "BlackHole not at index 1": Run `meet doctor --verbose` to see device indices, reconfigure with `meet init`
    - "Ollama not running": `ollama serve` in a separate terminal
    - "Notion token invalid": Create new integration at https://www.notion.so/my-integrations
    - "Empty transcript": Check audio routing — system audio must go through BlackHole
    - General: Run `meet doctor` for a full system health check
