# Phase 01 — UI Design Contract

> Visual and interaction contract for the GUI Foundation phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Stack note:** This is a PySide6 native desktop app (macOS, Apple Silicon). There is no
> web stack, no shadcn, no Tailwind. All visual constants live in `gui/theme.py` and are
> applied via a single `app.setStyleSheet(APP_STYLESHEET)` call. This contract is the
> canonical source of truth for what goes into `theme.py`.

## Spacing Scale

Declared values (all multiples of 4). Applied via `setContentsMargins` and `setSpacing`
on Qt layout managers — never via fixed `setGeometry` calls.

| Token | Value | QSS / Qt Usage |
|-------|-------|----------------|
| xs | 4px | Icon gaps, badge padding, status dot margin |
| sm | 8px | Button internal padding (top/bottom), inline element spacing |
| md | 16px | Default layout spacing, card internal padding |
| lg | 24px | Section padding (QGroupBox contents margin) |
| xl | 32px | Panel separation, major element gaps |
| 2xl | 48px | View-level top/bottom padding |
| 3xl | 64px | Not used in Phase 01 |

Exceptions:
- Sidebar item height: 40px (enforces comfortable click target on macOS)
- Window minimum size: 900 x 600px (from GUI-MILESTONE-PLAN.md — exact value required by spec)
- Window default size: 1100 x 700px (from GUI-MILESTONE-PLAN.md — exact value required by spec)
- Sidebar width: 180px fixed (standard macOS nav panel width)

## Typography

All sizes in points (Qt uses pt, not px, for font sizes). Line heights applied via
`QFontMetrics` spacing or QSS `line-height` where supported.

| Role | Font | Size | Weight | Line Height | QSS key |
|------|------|------|--------|-------------|---------|
| h1 — screen title | Helvetica Neue | 20pt | bold (700) | 1.2 | `font-h1` |
| h2 — section heading | Helvetica Neue | 13pt | bold (700) | 1.3 | `font-h2` |
| body — default UI text | Helvetica Neue | 11pt | normal (400) | 1.5 | `font-body` |
| mono — transcript, notes, SRT content, metadata, timestamps, captions | Menlo | 10pt | normal (400) | 1.6 | `font-mono` / `font-small` |

Source: GUI-MILESTONE-PLAN.md FONTS dict + CONTEXT.md specifics section.

Rules:
- Only `h1` and `h2` use bold weight. All other roles use normal (400).
- `font-small` is an alias for `font-mono` (10pt Menlo). Use it for metadata, timestamps,
  and captions. The same 10pt Menlo size covers both monospace content areas and small
  informational text.
- `mono` is ONLY for `QPlainTextEdit` content areas that display transcript, notes, or SRT
  text, plus small metadata labels. Never use Menlo for primary UI labels, h1/h2 headings,
  or primary action buttons.
- Never hardcode font family strings outside `theme.py`.

| Role | Value | Usage |
|------|-------|-------|

### Semantic Colors (10% accent + status)

| Role | Value | Reserved For |
|------|-------|--------------|
| Accent | `#0A84FF` | Active sidebar item highlight, primary action buttons ("Start Recording", "Save Settings"), selected state indicator on sidebar items. Never used for informational labels, icons, or decorative elements. |
| Green | `#30D158` | Pipeline step "done" fill (Recorded/Transcribed/Summarized/Notion indicators), health check OK card left border, recording active dot |
| Red / Destructive | `#FF3B30` | Record button (idle state), health check ERROR card left border, inline validation errors on template name field |
| Yellow / Warning | `#FF9F0A` | Health check WARNING card left border, built-in template read-only warning banner background tint |

Accent reserved for: active sidebar item, primary action buttons only. Do not use `#0A84FF` on health check cards, status badges, pipeline steps, or any informational display.

## Interaction Contract

### Navigation (Phase 01 core interaction)

| Trigger | Behavior | Qt mechanism |
|---------|----------|--------------|
| Sidebar item click | Switch QStackedWidget to corresponding index | `QListWidget.currentRowChanged` → `QStackedWidget.setCurrentIndex` |
| Sidebar item order | Dashboard(0), Sessions(1), Record(2), Templates(3), Settings(4), Health Check(5) | Index order matches addWidget order |
| Default selected item | Dashboard (index 0) on launch | `sidebar.setCurrentRow(0)` in MainWindow.__init__ |

### Placeholder View Contract (Phase 01 only)

Each of the 6 views is a placeholder for Phase 01. Each renders a centered `QLabel` with
the screen name in `h1` style and `text_primary` color. No data loading, no worker calls,
no buttons. This is deliberate — placeholder scope avoids premature work.

### Window Behavior

| Property | Value |
|----------|-------|
| Minimum size | 900 x 600px |
| Default size on launch | 1100 x 700px |
| HiDPI | `Qt.AA_UseHighDpiPixmaps` attribute set on QApplication |
| App name | "Meeting Notes" (`setApplicationName` + `setApplicationDisplayName`) |

## State Display Contract

Phase 01 has no stateful views (all placeholders). The following states are defined here
for Phase 02–04 implementation guidance — they must be consistent with this contract.

| State | Visual indicator | Color |
|-------|-----------------|-------|
| Recording active | Pulsing red circle button (opacity animation via QPropertyAnimation) | `#FF3B30` |
| Pipeline step complete | Filled circle + step label | `#30D158` |
| Pipeline step pending | Unfilled circle + step label | `#E5E5EA` border, `#F5F5F7` fill |
| Health check OK | Green left border (4px) on card | `#30D158` |
| Health check WARNING | Yellow left border (4px) on card | `#FF9F0A` |
| Health check ERROR | Red left border (4px) on card | `#FF3B30` |
| Built-in template (read-only) | Yellow banner above editor | `#FF9F0A` tint at 15% opacity, `#1C1C1E` text |
| Password field hidden | QLineEdit.Password echo mode | — |
| Password field revealed | QLineEdit.Normal echo mode | — |
