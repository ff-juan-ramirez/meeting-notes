# Phase 02 — UI Design Contract

> Visual and interaction contract for the Sessions & Dashboard phase.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Stack note:** This is a PySide6 native desktop app (macOS, Apple Silicon). There is no
> web stack, no shadcn, no Tailwind. All visual constants live in `gui/theme.py` and are
> applied via a single `app.setStyleSheet(APP_STYLESHEET)` call. This contract extends
> the Phase 01 design contract — all tokens carry forward unchanged unless explicitly
> overridden below.
>
> **Canonical references:** `GUI-MILESTONE-PLAN.md` (worker signals, threading rules, PySide6 standards)
> and `meeting-notes-ui-mockups.pdf` pages 1–2 (structural layout guide).

`setSpacing` on Qt layout managers — never via fixed `setGeometry` calls.

| Token | Value | QSS / Qt Usage |
|-------|-------|----------------|
| xs | 4px | Status dot margin, badge horizontal padding, pipeline arrow spacing |
| sm | 8px | Button internal padding (top/bottom), row widget internal spacing, filter pill gap |
| md | 16px | Default layout spacing, card internal padding, detail panel section padding |
| lg | 24px | Section heading margin, QGroupBox contents margin |
| xl | 32px | Panel separation (QSplitter section padding), major element gaps |

Exceptions (carried forward from Phase 01):
- Sidebar item height: 40px
- Window minimum size: 900 x 600px

- SessionRowWidget preferred height: 56px (accommodates two text lines + vertical padding at sm top/bottom)

- Dashboard stat card: 120px minimum width, 80px height, md (16px) internal padding
- Status pill height: 24px; padding: 4px 8px (xs vertical, sm horizontal)

## Typography

All sizes in points (Qt uses pt). Entire scale carries forward from Phase 01 unchanged.

| Role | Font | Size | Weight | Line Height | QSS key | Phase 02 usage |
|------|------|------|--------|-------------|---------|----------------|
| h1 — screen title | Helvetica Neue | 20pt | bold (700) | 1.2 | `font-h1` | "Sessions" / "Dashboard" view heading |
| h2 — section heading | Helvetica Neue | 13pt | bold (700) | 1.3 | `font-h2` | Detail panel section headers, stat card labels, "Recent Sessions" heading |
| body — default UI text | Helvetica Neue | 11pt | normal (400) | 1.5 | `font-body` | Session row title, date, duration; detail panel metadata fields; filter pill labels; button labels |
| mono / small — metadata, timestamps, content | Menlo | 10pt | normal (400) | 1.6 | `font-mono` / `font-small` | QPlainTextEdit content (transcript, notes, SRT); session row timestamp caption; stat count value; elapsed time in recording pill |

Rules (all carry forward from Phase 01):
- Only h1 and h2 use bold. All other roles use normal (400).
- `font-small` is an alias for `font-mono` (10pt Menlo). Use it for timestamps, captions, and metadata labels.
- Menlo is ONLY for QPlainTextEdit content areas and small metadata/timestamp labels. Never for primary UI labels, headings, or action buttons.
- Never hardcode font family strings outside `theme.py`.

Phase 02 specific rules:
- SessionRowWidget title line: `body` role (11pt Helvetica Neue normal).
- SessionRowWidget subtitle line (date + duration + status): `font-small` role (10pt Menlo).
- Dashboard stat count value (the number): `h1` role (20pt Helvetica Neue bold).
- Dashboard stat label below count: `font-small` role (10pt Menlo normal).
- Status pill label text: `font-small` role (10pt Menlo normal).
- Pipeline step label text: `font-small` role (10pt Menlo normal).

## Interaction Contract

### Sessions View

| Trigger | Behavior | Qt mechanism |
|---------|----------|--------------|
| Sessions view becomes visible | Load all sessions from disk via `list_sessions()` equivalent; populate session list | `showEvent` → `_refresh_sessions()` |
| Status filter change (All / Recorded / Transcribed / Summarized) | Filter in-memory session list; repopulate QListWidget immediately (no disk read) | `QComboBox.currentTextChanged` → `_apply_filter()` |
| Session row click | Populate detail panel with session metadata; set as current session | `QListWidget.currentItemChanged` → `_load_session_detail(session_id)` |
| Transcribe button click | Disable all action buttons + "Open Notion" button; show progress label; start TranscribeWorker | `QPushButton.clicked` → `_start_transcribe()` |
| Summarize button click | Disable all action buttons; show progress label; start SummarizeWorker | `QPushButton.clicked` → `_start_summarize()` |
| Worker progress signal | Update status label text below action buttons | `TranscribeWorker.progress(str)` → `_status_label.setText(str)` |
| Worker finished signal | Auto-refresh detail panel from disk; clear status label; re-enable buttons | `TranscribeWorker.finished(str, int)` → `_on_transcribe_done()` |
| Worker failed signal | Show `QMessageBox.warning()`; clear status label; re-enable buttons | `TranscribeWorker.failed(str)` → `_on_worker_failed()` |
| Open Notion button click | Open URL in default browser; no confirmation required | `QDesktopServices.openUrl(QUrl(notion_url))` |
| Tab switch (Transcript / Notes / SRT) | Show file content in QPlainTextEdit; read-only | `QTabWidget.currentChanged` → load file content |
| No session selected (initial state) | Right panel shows centered "Select a session to view details." No buttons rendered | Controlled by QStackedWidget or conditional visibility |
| select_session(session_id) called from MainWindow | Pre-select the matching row in QListWidget; populate detail panel | `SessionsView.select_session(str)` public method |

Double-start guard: before creating any worker, check `self._worker is None or not self._worker.isRunning()`. If worker is running, return early without starting a second.

Worker lifecycle: hold reference as `self._worker` on the owning widget. Connect `finished` → `self._worker.deleteLater`. Implement `closeEvent` to `quit()` + `wait()` on any running worker.

### Dashboard View

| Trigger | Behavior | Qt mechanism |
|---------|----------|--------------|
| Dashboard view becomes visible | Load aggregate stats + recent 5 sessions from disk; start QTimer if not running | `showEvent` → `_refresh_dashboard()` + `_poll_timer.start()` |
| QTimer fires (every 2 seconds) | Re-read `state.json`; update recording pill and button label | `QTimer(2000).timeout` → `_refresh_recording_state()` |
| Recording state: idle | Status pill background `#E5E5EA`, text "Idle"; button label "Start Recording" | Controlled by `_set_recording_ui(active=False)` |
| Recording state: active | Status pill background `#FF3B30`, text "● Recording • H:MM:SS"; button label "Go to Record" | Controlled by `_set_recording_ui(active=True, elapsed=N)` |
| "Start Recording" button click | Emit `navigate_requested(2)` signal; MainWindow switches to Record view | `DashboardView.navigate_requested.emit(2)` |
| "Go to Record" button click | Same as above — navigate to Record view; stop action is Record view's responsibility | `DashboardView.navigate_requested.emit(2)` |
| Recent session row click | Emit `session_selected(session_id)` signal; MainWindow navigates to Sessions and pre-selects row | `DashboardView.session_selected.emit(session_id)` → `MainWindow.navigate_to(1)` + `sessions_view.select_session(session_id)` |
| Dashboard view hidden | Optionally stop QTimer to avoid unnecessary polling | `hideEvent` → `_poll_timer.stop()` (optional optimization) |

Signals defined at class level on DashboardView:
- `navigate_requested = Signal(int)` — sidebar index to switch to
- `session_selected = Signal(str)` — session_id for Sessions pre-select

### State Display (Phase 02)

| State | Visual | Color |
|-------|--------|-------|
| Pipeline step: done | Filled 24×24px circle | `#30D158` (green) fill, no border |
| Pipeline step: pending | 24×24px circle with border | `#E5E5EA` border, `#F5F5F7` fill |
| Pipeline steps connector | `QLabel("→")` between circles | body role, text_primary color |
| Recording pill: idle | 24px tall pill | background `#E5E5EA`, text "Idle", `#1C1C1E` text |
| Recording pill: active | 24px tall pill | background `#FF3B30`, text "● Recording • H:MM:SS", `#FFFFFF` text |
| Action buttons: disabled (worker running) | All buttons grayed out | Qt built-in disabled appearance via `setEnabled(False)` |
| Status label: worker running | Visible QLabel below action buttons | body role, text_primary, text from `progress` signal |
| Status label: idle | Hidden / empty string | `status_label.hide()` or `status_label.setText("")` |

## State Machine Summary (Sessions Detail Panel)

The detail panel operates in two modes controlled by whether a session is selected:

**No session selected (initial / unselected):**
- Show centered "Select a session to view details." QLabel
- No action buttons rendered
- No tab widget rendered

**Session selected:**
- Show session metadata (title, date, duration, word count, speaker count)
- Show pipeline step indicator (4 steps)
- Show action buttons row: "Transcribe" (enabled if wav exists and not transcribed), "Summarize" (enabled if transcribed and not summarized), "Open in Notion" (enabled if notion_url set)
- Show status label (hidden when idle, visible during worker run)
- Show QTabWidget with Transcript / Notes / SRT tabs, each `QPlainTextEdit(ReadOnly=True)`

**Worker running:**
- All three action buttons disabled via `setEnabled(False)`
- Status label visible with progress text from worker signal
- Double-start guard active
