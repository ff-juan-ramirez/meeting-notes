`showEvent(self, event)` — calls `_refresh_sessions()`. Also lazily populates template combo if empty:
```python
if self._template_combo.count() == 0:
```

`_refresh_sessions(self)` — scan `metadata_dir.glob("*.json")`, sorted by `_sort_key` (newest first). Build list of session dicts with keys: session_id, title, date, duration_display, duration_seconds, status, wav_path, transcript_path, notes_path, notion_url, srt_path, recording_name, word_count, speaker_turns. Store as `self._sessions`. Call `_apply_filter()`.

`_apply_filter(self)` — filter `self._sessions` by status combo. Map: "All" -> show all, "Recorded" -> "not-transcribed", "Transcribed" -> "transcribed", "Summarized" -> "summarized". Clear and repopulate `self._list_widget` with `SessionRowWidget` instances. Use `QListWidgetItem` + `setItemWidget` + `item.setSizeHint(widget.sizeHint())` per Pitfall 7. If filtered list is empty, show empty state (stack index 1); else show list (stack index 0).

`_on_session_selected(self, row: int)` — if row < 0, show empty detail (stack index 0). Otherwise get session dict from filtered list, store as `self._current_session`, show detail panel (stack index 1), call `_populate_detail()`.

`_populate_detail(self)` — populate all metadata labels, pipeline circles, button states, and tab content from `self._current_session`:
- Pipeline steps: check wav_path exists (Recorded), transcript_path exists (Transcribed), notes_path exists (Summarized), notion_url truthy (Notion). Set each circle's style property accordingly.
- Button states per D-01: Transcribe enabled if wav exists and NOT transcribed. Summarize enabled if transcribed and template selected. Open Notion enabled if notion_url truthy.
- Tab content: read file at transcript_path / notes_path / srt_path. If file doesn't exist, set placeholder text ("Not yet transcribed." / "Not yet summarized." / "No SRT file.") per UI-SPEC copywriting.
- SRT path: derive as `transcript_path.replace('.txt', '.srt')` if transcript_path exists, or check metadata for `srt_path` key.

`_start_transcribe(self)` — per D-01: double-start guard (`if self._worker and self._worker.isRunning(): return`). Get wav_path from current session. Create `TranscribeWorker(Path(wav_path), self._config)`. Connect signals: `progress` -> `_on_progress`, `finished` -> `_on_transcribe_done`, `failed` -> `_on_worker_failed`, `finished` -> `self._worker.deleteLater`. Call `_set_buttons_enabled(False)`. `self._status_label.show()`. `self._worker.start()`.

`_start_summarize(self)` — per D-01: double-start guard. Get stem, template name from combo, title from line edit (or None if empty). Create `SummarizeWorker(stem, template, title, self._config)`. Connect signals same pattern. Disable buttons. Show status label. Start worker.

`_on_progress(self, msg: str)` — `self._status_label.setText(msg); self._status_label.show()`

`_on_transcribe_done(self, stem: str, word_count: int)` — per D-02 and Pitfall 8: check if `stem == self._current_session.get("session_id")`. `_on_summarize_done(self, notion_url: str)` — per D-02: refresh detail from disk. Clear status label. Re-enable buttons. Refresh sessions list.

`_on_worker_failed(self, error: str)` — per D-03: `QMessageBox.warning(self, "Operation Failed", f"{error}. Check your configuration and try again.")`. Clear status label. Re-enable buttons.
