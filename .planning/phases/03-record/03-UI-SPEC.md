## Interaction Contract

### State Machine (Phase 03 core interaction)

The RecordView has exactly three states: **Idle**, **Recording**, and **Stopping**.
All state transitions are driven by worker signals. Direct state mutation from button clicks
is not permitted ‚Äî buttons only start workers.

| Trigger | From State | To State | Qt mechanism |
|---------|-----------|---------|--------------|
| User clicks Record button | Idle | Recording (pending) | `RecordWorker.start()` ‚Üí wait for `started` signal |
| `RecordWorker.started` fires | Idle (pending) | Recording | `@Slot(int, str)` on view |
| `RecordWorker.failed` fires | Idle (pending) | Idle | `QMessageBox.warning()` ‚Üí reset to Idle |
| User clicks Stop button | Recording | Stopping | `StopWorker.start()` ‚Üí wait for `stopped` signal |
| `StopWorker.stopped` fires | Stopping | Idle | `@Slot(str)` on view ‚Üí refresh recent list, clear title |
| `StopWorker.failed` fires | Stopping | Recording | `QMessageBox.warning()` ‚Üí return to Recording (D-04) |

Source: CONTEXT.md D-01 through D-05 ‚Äî all state transitions are locked decisions.

### Idle State Layout

| Element | Widget | Detail |

### showEvent Behavior

On every `showEvent`, RecordView must:
1. Check `state.json` via `read_state()`. If an active recording exists (pid + output_path), restore to Recording state with elapsed time calculated from the stored start_time.
2. If no active recording, ensure Idle state ‚Äî title cleared, animation stopped, recent list refreshed.
3. Refresh recent recordings list by calling `list_sessions()` and taking the 2 most recent entries sorted by `start_time` descending.

Source: CONTEXT.md Existing Code Insights (showEvent pattern from Phase 02).

## Animation Contract

Phase 03 introduces one animation. No other views use animation.

| Animation | Widget | Property | Duration | From | To | Loop | Trigger |
|-----------|--------|----------|----------|------|----|------|---------|
| Record button pulse | `QPushButton` (record/stop button) | `windowOpacity` via `QPropertyAnimation` on the button's `opacity` | 1200ms | 1.0 | 0.4 | Infinite (`setLoopCount(-1)`) | Starts when Recording state entered; stops and resets to 1.0 when Stopping state entered |

Implementation note: `QPropertyAnimation` targets the `opacity` property on the button widget
directly. The `QEasingCurve.InOutSine` easing curve produces a smooth pulse. Stop the animation
by calling `animation.stop()` and then `button.setWindowOpacity(1.0)` to reset.

Source: CONTEXT.md Claude's Discretion ‚Äî "QPropertyAnimation opacity pulse on the record button
in Recording state (per milestone plan) ‚Äî implementation details (duration, opacity range)
left to Claude."

## Copywriting Contract

| Element | Copy | Notes |
|---------|------|-------|
| Primary CTA (Idle) | "Record" | Button label. Verb only ‚Äî no noun needed on a circular button. |
| Primary CTA (Recording) | "Stop" | Same button, label swaps. Present tense imperative. |
| Title field placeholder | "Recording title (optional)" | Lowercase "optional" in parentheses. No period. |
| Device info label | "System: :{n} \| Mic: :{n}" | Exact format e.g. "System: :1 \| Mic: :2". Vertical bar with single space each side. |
| Status label ‚Äî Recording | "Recording ‚Äî capturing system audio + microphone" | Em dash (‚Äî) between "Recording" and description. No period. Body role. |
| Status label ‚Äî Stopping | "Stopping and saving..." | Ellipsis (‚Ä¶) as Unicode character U+2026, not three dots. No period after. Body role. |
| Elapsed time | "0:00:00" ‚Üí "H:MM:SS" | Zero-padded minutes and seconds; hours not zero-padded. Leading "0:" for sub-one-hour recordings. |
| "Recent Recordings" section label | "Recent Recordings" | Title case. h2 role. Shown only when ‚â•1 session exists. |
| Empty recent recordings | (section hidden entirely) | Do not show "No recent recordings" ‚Äî simply hide the section when sessions list is empty. |
| RecordWorker failure dialog | Title: "Recording Failed" / Body: "Could not start recording: {error}. Check your audio device configuration." | `QMessageBox.warning()`. "Check" is the action verb. |
| StopWorker failure dialog | Title: "Stop Failed" / Body: "Could not save the recording: {error}. The recording may still be active ‚Äî try stopping again or restart the app." | `QMessageBox.warning()`. View returns to Recording state (D-04). |
